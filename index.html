一、文档说明

本文档定义“中医药优势病种证据系统的智能化构建和应用示范”项目中所需的软件系统接口规范。

二、适用对象说明

  • 本规范可用于开发与智能证据系统的对接的应用程序。
  • 本规范适用于向智能证据系统注入或者查询文本数据结果。
  • 本规范所定义的接口，均采用REST基础的接口规范，所有请求统一通过HTTPS传输。

三、名词解释

可选字段：在向数据库注入条目时，该字段可以不存在

必须字段：在向数据库注入条目时，该字段必须存在，否则将被拒绝

appid：智能证据系统为接入的应用提供的唯一标识

appkey：智能证据系统为接入的应用提供的共享密钥

appsecret：智能证据系统为接入的应用提供的密码

...

四、系统框图

本文档定义如图所示接口。具体接口规范，将在下一部分中详述。

  • 接口1: 原始数据库注入接口 该接口用于向原始数据库注入数据库条目。
  • 接口2: 原始数据库检索接口 该接口用于查询原始数据库中的条目。
  • 接口3: 证据数据库检索接口 该接口用于查询证据库中的条目。

五、数据字段

1.条目总体结构

一个数据库条目，应该包括如下内容：一般信息(general_info)、文献基本数据(bib_info)、临床资料(clinic_info)、分组信息(grouping_info)、干预措施(interventions)、结局指标(results)等。而每一项内容又包含若干个字段。具体将在下文详述。

一个条目的JSON示例： 

{
	general_info:
	{
		id: 10001,
		name: “一篇临床文献”,
		status: 1,
		create_time: 2019-01-02,
		edit_time: 2020-02-10,
		editor: “author1”,
		reviewer: “reviewer1”
	},
	bib_info:{
		…
	},
	clinic_info:{
		…
	},
	Trial_design:{
		…
	},
	grouping_info:{
		…
	},
	interventions: {
		#array of intervention
	},
	results:{ 
		#array of results
	}
	abnormal_events_int:{
		#array of abnormal_event
	}
	abnormal_events_ctr:{
		#array of abnormal_event
	}
}
